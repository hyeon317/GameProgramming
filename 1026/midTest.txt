2. 함수 포인터에 대해 예와 함께 서술하시오

함수포인터는 변수가 아닌 함수를 가리키는 포인터로, callback을 구현할 때 함수를 또 다른 함수의 인자로 넘겨주거나, 여러 함수를 배열로 관리하고자 할 때 사용된다.
함수포인터도 포인터이기 때문에, 일반적인 포인터와 같이 메모리의 주소를 가리키지만, 일반적인 포인터와 달리 데이터를 가리키는 것이 아니라 코드의 위치를 가리킨다.
함수포인터로는 메모리의 할당과 회수가 불가능하며, 그렇기 때문에 malloc(), free()를 사용할 수 없다.

#include <stdio.h>
void fun(int num) {
	printf("%d\n", num);
}
int main() {
	void (*fun_ptr)(int);
	fun_ptr = &fun;
	(*fun_ptr)(3);
	return 0;
}

5. CSS의 Selector에 대해 간단한 예와 함께 서술하시오

CSS는 HTML의 스타일을 정의한다.
스타일을 정의할 HTML 요소를 특정할 필요가 있는데, 그 목적으로 사용되는 것이 Selector이다.
즉, Selector로 스타일을 정의할 HTML요소를 특정하고, 그 요소에 스타일을 정의한다.
예를 들어, h1{color:blue, font-size:10px};에서 h1은 태그 이름 셀렉터이며,
그 외에도 .으로 시작하는 class 셀렉터, #으로 시작하는 id 셀렉터, >으로 조합되는 자식 셀렉터, *으로 표시되는 전체 셀렉터 등이 있다.
